Animation Maker Testing Guide

Test Steps

  1. Upload Test Files:
    • Field Image: test-field.svg
    • Robot Sprite: test-robot.svg
  2. Create Animation:
    • Click on the field to add at least 2 waypoints
    • Adjust headings and add actions as desired
  3. Test Recording:
    • Click "Record" to start recording
    • Animation should start automatically
    • Press SPACE to pause and test pan/zoom
    • Click "Stop Recording" when done
  4. Test Export:
    • Modal should appear asking for filename
    • Try "Save as WebM" - should download WebM file
    • Try "Save as GIF" - should convert and download GIF

Expected Behavior

✓ WebM Export: Should work reliably with browser MediaRecorder API
⚠ GIF Conversion: Will try FFmpeg.wasm first, then fallback to Cloudinary if configured
✗ Known Issues: FFmpeg.wasm requires large download (~32MB) and may be slow

Quick Access

Open Animation Maker

Troubleshooting

  • GIF conversion fails: Check browser console for FFmpeg errors. Cloudinary fallback should work if configured.
  • Recording doesn't start: Ensure browser supports MediaRecorder API (Chrome, Firefox, Safari)
  • Files won't upload: Check file formats - images (.jpg, .png, .svg) and PDFs are supported
